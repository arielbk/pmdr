## `position-persistence` — 2026-05-23 14:37:19

**Status:** done
**Summary:** Integrated `FloatingTimerPosition` into `FloatingTimerPanelController` so showing the panel uses the active display's saved origin or that display's default top-right origin, while hiding saves the current frame origin. `AppDelegate` now saves the visible panel position during app termination so quit/relaunch preserves the last position even if the user quits without hiding first.
**Deviations:** Codex marked `needs-review` after its in-sandbox test run reported 76 passing; on the host machine `xcrun xctest` reproduces a real failure (`FloatingTimerPanelControllerTests.testHideRecordsCurrentPositionForPanelDisplay`: expected `(333, 444)`, got `nil`). Root cause is in `FloatingTimerPanelController.saveCurrentPosition()` — it calls `screen(containing: panel.frame) ?? screenProvider()`, where `screen(containing:)` scans the real `NSScreen.screens` rather than the injected `screenProvider`. When a real desktop screen contains the panel rectangle, the position is recorded against the real display ID instead of the test's `TestScreen` ID, so `store.position(for: testScreen)` returns nil. The same logic will also misattribute saves on a real machine when the panel sits on a non-primary screen whose `displayID` does not equal `screenProvider()`'s — the persistence will go to the geometric display but lookups use the active screen.
**Suggested fix:** make `saveCurrentPosition()` prefer the injected `screenProvider()` (the explicit "active display" signal), and only fall back to `screen(containing:)` when the provider returns nil. The unit test then exercises the same code path as production. Re-run `xcodegen generate && xcodebuild build-for-testing -scheme pmdr-menubar -destination 'platform=macOS' -derivedDataPath DerivedData CODE_SIGNING_ALLOWED=NO && xcrun xctest DerivedData/Build/Products/Debug/pmdr-menubarTests.xctest` from `apps/menubar/`; all 76 tests should pass.
**Handoff:** The controller now accepts injectable `positionStore` and `screenProvider` for tests; production uses `NSScreen.main ?? NSScreen.screens.first` as the active display. `AppDelegate.applicationWillTerminate(_:)` flushes the visible panel's position on quit. `xcodegen generate` succeeds; `xcodebuild build-for-testing` succeeds. Direct `xcrun xctest DerivedData/Build/Products/Debug/pmdr-menubarTests.xctest` reports **75 passing, 1 failing, 3 skipped** on the host.
